recordTriggeredByCustomHandling

プロパティ 
public var recordTriggeredByCustomHandling:Boolean

言語バージョン: ActionScript 3.0
製品バージョン: Flex 3
ランタイムバージョン: Flash Player 9, AIR 1.1

このイベントの現在のレコードがカスタムレコードイベントから生じている場合は true、そうでない場合は false が含まれます。このフィールドを使用すると、フレームワークでトリガーされたレコードイベントとカスタムレコードとを区別できます。例えば、リストに select イベントがあり、現在、インデックスまたは選択されたアイテムを使用して、選択の詳細が記録されている場合です。しかし、ユーザーは、両方を使用して詳細を記録する場合もありますが、同じイベント名で記録する場合もあります。フレームワークでトリガーされた select イベントと、Automation Manager で recordCustomAutomationEvent() API を使用してトリガーされた select イベントとを区別することはかなり面倒です。recordCustomAutomationEvent() API を使用してイベントがトリガーされた場合、このフラグは true になります。それ以外のすべての場合、フレームワークは、このフラグをデフォルト値で維持します。

言語バージョン: ActionScript 3.0
製品バージョン: Flex 3
ランタイムバージョン: Flash Player 9, AIR 1.1

AutomationRecordEvent.RECORD 定数により type プロパティ値が定義されます。これは record イベントのイベントオブジェクトのプロパティ値です。

イベントオブジェクトの各プロパティには次の値が設定されています。

プロパティ値
argsメソッドのパラメーターの配列です。
automationObject操作を送出する UIComponent を委譲します。
bubblestrue
cacheableイベントをイベントキャッシュに保存する場合は true、保存しない場合は false です。
cancelabletrue
currentTargetイベント処理を行うイベントリスナーの定義用オブジェクトを表します。 例えば、イベントリスナーの登録に myButton.addEventListener() を使用すると、myButton の値は currentTarget となります。
methodName操作の表示可能な名前です。
replayableEvent操作を表す基のイベントです。
targetイベントを送出したオブジェクトを表します。このオブジェクトは必ずしもイベントを受け取るオブジェクトとは限りません。 イベントを受け取るオブジェクトに常にアクセスするには、currentTarget プロパティを使用してください。
