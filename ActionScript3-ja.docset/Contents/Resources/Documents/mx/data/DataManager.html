DataManager クラスは、特定のデータ型の全インスタンスに対して、サーバー上のオブジェクトとの同期機能を提供します。DataManager は、同期、複製、競合検出、変更追跡、臨時的に接続されるサービスなどを提供します。

DataManager クラスは抽象クラスです。通常は DataManager を拡張する DataService インスタンスを構築するか、1 つまたは複数の RPCDataManager インスタンスを RPC サービスコンポーネント（HTTPMultiService、WebService、RemoteObject インスタンスなど）と併せて構築します。

Single Managed Object（SMO）を取得するには、DataManager.getItem() メソッドおよび DataManager.createItem() メソッドを使用します。

内容設定済みの ArrayCollection が不要になったとき、またはそのコレクション内のアイテムに対する更新の受信および作成が不要になったときは、DataManager.releaseCollection() メソッドを呼び出します。

DataManager.getItem() または DataManager.createItem() を使用して前に取得したアイテムが不要になった場合、または更新を受信または作成しない場合、DataManager.releaseItem() メソッドを呼び出します。DataManager.createItem() メソッドと DataManager.getItem() メソッドは ItemReference インスタンスを返します。ItemReference は Single Managed Object（SMO）の特定のインスタンスを解放するときに使用します。DataManager.releaseItem() メソッドとは異なり、ItemReference.releaseItem() メソッドを呼び出すと Single Managed Object（SMO）の特定のインスタンスが解放されます。DataManager.releaseItem() メソッドは、指定した ID と一致する ID で検出された最初の Single Managed Object（SMO）を解放します。これは、DataManager.getItem() メソッドおよび DataManager.createItem() メソッドの呼び出しの数が DataManager.releaseItem() メソッドの呼び出しの数と一致している限り問題ありません。

接続を必要とするメソッドが初めて呼び出されると、リモート宛先への接続が自動的に作成されます。例えば、未コミットの更新があるときに DataManager.fill() メソッドまたは DataManager.commit() メソッドを呼び出すと、DataManager インスタンスはリモート宛先への接続を確立しようとします。DataManager インスタンスの接続を強制的に解除するには、DataManager.disconnect() メソッドを呼び出します。不要になったリモート宛先リソースをクリーンアップするには、DataManager.disconnect() メソッドを使用できます。他のサービスで同じ物理接続が使用されている可能性があるため、DataManager.disconnect() メソッドを呼び出すと接続が終了する場合があります。

バインド可能なプロパティ connected は、関連付けられている DataManager 接続の現在の状態を示します。

各 DataManager インスタンスには mx.data.DataStore 型のオブジェクトを返す dataStore プロパティがあります。このオブジェクトには、1 つ以上の DataManager インスタンスに関する未コミットの変更が格納されています。デフォルトでは、DataManager インスタンスは、管理対象の関連付けプロパティを持ち、同じチャネルセットを共有している他のデータサービスと同じ DataManager を共有します。DataStore インスタンスを共有しているときに、1 つの DataManager インスタンス上で commit() メソッドを呼び出すと、共有 DataStore インスタンスを使用しているすべての DataManager インスタンスが変更されます。関連付けられている DataManager インスタンスに加えられた変更の間に依存関係がある場合、操作の順序は維持されます。

DataManager と DataStore は、リモート呼び出しを必要とするすべての操作の結果およびフォルトイベントを送出します。メモ：同じ宛先をポイントする 2 つの DataManager インスタンスを作成した場合、これらのインスタンスは同じ管理対象オブジェクトのコレクションを共有し、同じイベントを送出します。例えば、1 つの DataManager インスタンスに宛先としてイベントリスナーを追加し、別のインスタンスを使用して操作を開始した場合、最初の DataManager インスタンスは引き続きそれらのイベントを受け取ります。DataManager に対して操作を開始するたびに、AsyncToken が返されます。そのトークンにイベントの応答側を追加すると、その操作に固有の結果イベントを受信できます。一般に、この方法を使用すると、特定の宛先に対して結果またはフォルトイベントハンドラーをグローバルに追加するよりもコードが簡潔になります。

DataManager インスタンスの管理対象オブジェクトのすべてのプロパティでデータバインディングを使用できます。管理対象オブジェクトに対してリモート操作が発生すると、DataManager インスタンスは「メッセージ」イベントも送出します。例えば、リモートプロセスでローカルで管理されているアイテムを更新すると、そのアイテムを管理しているすべての DataManager インスタンスに対してメッセージがプッシュされます。このメッセージは、MessageEvent.MESSAGE イベントとして送出されます。

リモート宛先で DataManager インスタンスがコミットする変更に対する応答として、またはローカルの未コミットの変更と競合するプッシュされた変更によって、競合イベントは送出されます。DataConflictEvent.CONFLICT をリスニングすることで競合イベントを処理できます。DataManager.conflicts.resolved プロパティにバインディングすることで、データバインディングを使用してユーザーインターフェイスコントロールを更新します。

DataManager クラスは、次のデータ同期設定モードをサポートしています。

  • autoCommit - true の場合、変更が検出されるたびに、各変更が直ちにリモート宛先へ送信されます。これが false の場合、commit() メソッドを明示的に呼び出す必要があります。未コミットの変更があるかどうかを判断するには、バインド可能な DataManager.commitRequired プロパティを使用します。
  • autoSyncEnabled - これは fill()、getItem()、createItem() の各メソッドの呼び出しに適用され、他のクライアントによってこれらのオブジェクトに加えられた変更、またはリモート宛先で DataServiceTransaction サーバープッシュ API を使用して加えられた変更を、管理対象インスタンスでリスニングするかどうかを決定します。
  • autoMerge プロパティ - リモート宛先から受信した変更を直ちに適用するか、DataManager.merge() メソッドの呼び出しが行われるまで待機するかを制御します。

特定のアイテムまたはすべてのアイテムに対する変更を元に戻すには、DataManager.revertChanges() メソッドを使用します。リモート宛先の操作でエラーが発生すると、リモート宛先のデータとの同期を維持するために、コミット済みの変更が未コミットのキューに戻されます。変更がエラーになった場合、通常は、フォルトハンドラーの一部として DataManager.revertChanges() メソッドを呼び出して変更を取り消します。

commit

()メソッド 
public function commit(itemsOrCollections:Array = null, cascadeCommit:Boolean = false):mx.rpc:AsyncToken

言語バージョン: ActionScript 3.0
製品バージョン: Adobe Digital Enterprise Platform Data Services for Java EE 3
ランタイムバージョン: Flash Player 9, AIR 1.1

この DataManager インスタンスに関連付けられている DataStore インスタンスによって現在管理されているすべてのコレクションに対する保留中の変更をコミットします。このメソッドを呼び出すことは、dataStore.commit() メソッドを呼び出すことと同等です。

autoCommit プロパティが true であるときはトランザクションが使用されないので、このメソッドを呼び出す必要はありません。トランザクションを使用するには、autoCommit プロパティを false に設定し、このメソッドを直接呼び出して変更のバッチをコミットします。

通常は、引数を指定せずに「commit()」を呼び出し、保留状態のすべての変更をコミットします。オプションの引数を指定すると、保留状態の変更の一部のみをコミットできます。DataStore 内のアイテムに対する変更をコミットするには、先に未解決の競合をすべて解決する必要があります。

アセンブラーで use-transactions が true に設定されている場合は、すべての変更がアトミックにコミットされます。バッチで指定された操作が 1 つでも失敗した場合は、すべての操作が適用されません。use-transactions が false に設定されている場合は、障害が発生するまで各操作が個別にコミットされ、障害が発生した時点で処理が停止します。エラーの前に発生した変更はすべて適用され、エラーの後に発生した変更は適用されません。バッチ処理でエラーが発生した場合は、すべての変更が未コミットのキューに戻されます。エラーの原因になった変更を削除する場合は、revertChanges を呼び出す必要があります。

デフォルトでは、commit メソッドを呼び出すと、この DataStore に対して作成、変更、および削除されたすべてのアイテムに対する保留状態の変更がすべてコミットされます。これらの変更の一部のみをコミットする場合は、バッチに含める変更を定義する管理対象 ArrayCollection インスタンスと管理対象アイテムの組み合わせを指定できます。指定した配列コレクションごとに、その配列コレクション内のアイテムのメンバーシップまたは順序に対する変更、またはその配列コレクション内のアイテムに対する変更がすべてコミットされます。また、個々の管理対象アイテムのリストを指定して、それらのアイテムに対する変更のみをコミットすることもできます。管理対象のコレクションまたはアイテムではないオブジェクトを itemsOrCollections パラメーターで指定すると、DataServiceError がスローされます。

itemsOrCollections パラメーターを使用していて、管理対象アイテムが他のアイテムを参照する関連付けパラメーターがある場合は、cascadeCommit パラメーターを使用して、関連付けられているアイテムに対する変更もバッチに含めるかどうかを制御できます。例えば、items パラメーターで Group アイテムを指定し、Group インスタンスに Person インスタンスのリストを参照する members プロパティがある場合、cascadeCommit を true にすると、Person インスタンスに対する変更もコミットされる変更のセットに含まれます。cascadeCommit が false の場合は、Group に対する変更のみが含まれます。従属するすべての変更をバッチに含めるには、cascadeCommit=true を使用します。

パラメーター

itemsOrCollections:Array (default = null) — これはオプションのパラメーターであり、保留状態のすべての変更をコミットするときはデフォルトで null になります。保留状態の変更の一部のみをコミットする場合は、この引数を使用して、管理する ListCollectionView インスタンスと管理するアイテムのリストを指定します。ほとんど場合、ListCollectionView オブジェクトは、fill メソッドに渡した ArrayCollection です。このメソッドに適したアイテムは、各アイテムの管理対象バージョンです。これらは、getItem または createItem から取得されるアイテム、または getItemAt メソッドを使用して管理対象コレクションから取得されるアイテムです。この配列内のいずれかの値で定義されているアイテムの変更のみがコミットされます。
 
cascadeCommit:Boolean (default = false) — true の場合は、このリストで指定された関連アイテムに対する変更もコミットされます。

戻り値
mx.rpc:AsyncToken — ResultEvent.RESULT または FaultEvent.FAULT の call プロパティで返される AsyncToken です。このオブジェクトにはカスタムデータを添付することができ、そのデータは後のイベント処理段階で検査できます。関連するアイテムが変更されていない場合は、AsyncToken の代わりに null が返されます。

例外
Error — 初期化が進行中の場合。これは、ランタイムの宛先が参照されていて、このサービスが初期化される前にコミットが呼び出された場合、または cacheID が設定されていて、ローカルキャッシュがロードされていない場合に、発生する可能性があります。どちらの場合も、commit を呼び出す前に、初期化の呼び出しまたは以前の操作から返されるトークンの結果またはフォルトを待つことで、エラーを防止できます。

revertChanges

言語バージョン: ActionScript 3.0

指定したアイテムに対する未コミットの変更を元に戻します。アイテムが指定されていない場合、未コミットの変更はすべて元に戻されます。作成操作を元に戻すと、アイテムが削除されます。削除操作を元に戻すと、アイテムが再び追加されます。更新操作を元に戻すと、アイテムのプロパティが変更を開始する前の元の状態に戻されます。このメソッドを呼び出すことは、dataStore.revertChanges() メソッドを呼び出すことと同等です。

このメソッドは、フォルトの発生後などに使用します。その場合、失敗した変更はすべて、未コミットの変更のリストに戻されます。その時点で、それらの変更を修正して問題を解決するか、このメソッドを使用して変更を元に戻すことで、それらの変更が再び送信されないようにすることができます。

アイテムが指定されていて、そのアイテムが変更されていない場合、このメソッドは false を返します。アイテムが指定されておらず、変更がない場合、このメソッドは false を返します。それ以外の場合は、true を返します。

item:IManaged (default = null) — 元に戻す変更への参照です。すべての変更を元に戻す場合は null を指定します。

戻り値
Boolean — 元に戻された変更があるかどうかを示すブール値です。
