パッケージmx.data
クラスpublic class RPCDataServiceAdapter
継承RPCDataServiceAdapter DataServiceAdapter Object

言語バージョン: ActionScript 3.0
製品バージョン: Adobe Digital Enterprise Platform Data Services for Java EE 3
ランタイムバージョン: Flash Player 9, AIR 1.1

これは、DataService 実装の新しい型を実装するために使用する抽象基本クラスです。このクラスを拡張し、createItem や updateItem などのメソッドをオーバーライドして、それらの呼び出しの実装方法をカスタマイズします。DataManagement は、必要に応じてこれらのメソッドを自動的に呼び出します。

このサービスを使用する上で最も難しい部分は、データ管理の追加によるユーザーコードへの影響が最小限になるように、既存のサービスの呼び出しを引き継ぐことです。サービスの呼び出しは同じように動作し、同じイベントとトークンが発行されるので、管理する必要があるのは結果のみです。これを行うために、RPCDataServiceAdapter は ManagedOperation を標準の RPC サービス操作に関連付けます。初期化時に操作の operationManager プロパティが設定され、エンドユーザーの呼び出しはこのクラスのプロキシメソッド（queryProxy など）にリダイレクトされます。これにより、対応するデータ管理呼び出しが生成され、管理対象トークンがエンドユーザーに返されて、エンドユーザーは管理対象操作の結果を取得できます。 


 order is:
  user calls service method
     calls service delegates to operationManager - ManagedOpertion.xxProxy methods
        calls managerOperation kicks off DMS op --> DataManager.xx
           calls DMS op calls adapter to do the op --> DataServiceAdapter.invoke
              calls: RPCDataServiceAdapter invokes the service operation and sets the "raw" lastResult
              returns: adapter's result comes back in and is converted to DMS Result Message - RPCDataServiceResult.xxResult
        returns: data manager processes result - updates managed object --> DataListRequestResponder/CommitResponder
     returns: service is updated to lastResult from the operation
  returns: user's result event is called

checkForConflict

()メソッド
protected function checkForConflict(resultEvent:ResultEvent, token:mx.rpc:AsyncToken):void

言語バージョン: ActionScript 3.0
製品バージョン: Adobe Digital Enterprise Platform Data Services for Java EE 3
ランタイムバージョン: Flash Player 9, AIR 1.1

このメソッドは、競合の可能性を示すフォルトが返された後に呼び出されます。そのフォルトのフォルトハンドラーで、getItem() を呼び出してサーバー上のアイテムの最新バージョンを取得します。

このメソッドをオーバーライドする必要があるのは、例えば、競合が検出されなかった後でアクションを実行する必要がある場合です。

パラメーター

resultEvent:ResultEvent — getItem() の呼び出しからのイベントオブジェクトです。
 
token:mx.rpc:AsyncToken — cause プロパティが含まれます。このプロパティは、変更の元の DataMessage です。このメソッドは、競合があるときは Conflicts.raiseConflict() を呼び出すことができ、競合がないと判断された場合は何も行いません。

このメソッドは、createItem 操作を実行する必要がある場合に RPCDataServiceAdapter によって呼び出されます。デフォルトの実装では、createItem 操作に対して登録されている管理対象操作が、そのアイテムを 1 番目のパラメーターとして使用して呼び出されます。複数のサービスを呼び出す必要がある場合、またはこの操作のパラメーター署名を変更する必要がある場合は、RPCDataServiceAdapter を拡張して、この createItem メソッドをオーバーライドできます。

パラメーター

dataManager:RPCDataManager — 呼び出される作成操作の制御でのマネージャーです。
 
item:Object — 作成するアイテムです。

戻り値
mx.rpc:AsyncToken — です。このトークンを使用すると、この作成操作の結果またはフォルトイベントを受け取るためのコールバック関数を登録できます。このトークンは、ResultEvent.RESULT または FaultEvent.FAULT の call プロパティでも返されます。このオブジェクトにはカスタムデータを添付することができ、そのデータは後のイベント処理段階で検査できます。

handleFault

()メソッド 
public function handleFault(errMsg:ErrorMessage, cause:DataMessage):void

言語バージョン: ActionScript 3.0
製品バージョン: Adobe Digital Enterprise Platform Data Services for Java EE 3
ランタイムバージョン: Flash Player 9, AIR 1.1

これは、コレクションの作成、更新、削除、または更新操作のフォルトが処理された後に呼び出されます。これを使用すると、エラー処理と競合検査に関連付けられたロジックをアダプターでカスタマイズできる場合があります。デフォルトの実装では、RPCDataManager conflictMode 機能が実装されます。UPDATE 操作または DELETE 操作のフォルトを受け取ると、現在のサーバーバージョンと、更新または削除されたオブジェクトの元のバージョンが比較されます。サーバーバージョンが異なる場合は、競合として扱われます。この場合、フォルトの原因になった変更は既に、未コミットの変更バッチに戻されているものと仮定されます。

パラメーター

errMsg:ErrorMessage — エラーの説明です。
 
cause:DataMessage — エラーの原因になった操作です。
