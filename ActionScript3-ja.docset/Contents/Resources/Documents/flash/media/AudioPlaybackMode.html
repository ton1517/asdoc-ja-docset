AudioPlaybackMode クラスは、SoundMixer クラスの audioPlaybackMode プロパティの定数を定義します。

これらの各定数は、特定の用途に応じたモバイルでのオーディオの一連の動作を表します。

AMBIENT

定数
public static const AMBIENT:String = "ambient"

言語バージョン: ActionScript 3.0
ランタイムバージョン: AIR 3.4

アンビエントサウンドを再生するためのモード。アプリケーションの実際の目的に対する伴奏のサウンドおよびアプリケーションの正常な使用に必要でないサウンドの再生にこのモードを使用します。このモードを使用すると、オーディオはスピーカーフォン経由で出力されます。ヘッドホンが接続されている場合は、スピーカーフォンが無効になります。オーディオの動作と優先度を決定する場合、オペレーティングシステムではアンビエントサウンドのデフォルト値を使用します。

この設定がデスクトップまたはテレビに特別な影響を与えることはありません。

iOS では、このモードを使用すると、画面をロックして（または他の方法でアプリケーションがバックグラウンドに送られて）オーディオが無音になり、マイクが使用されていない間、ハードウェアのサイレントスイッチを有効にします。マイクが使用されている場合は、オーディオはこれらのアクションの影響を受けません。

注意：ヘッドセットが接続されている場合、サイレントスイッチではオーディオが無音になりません。

    

MEDIA

定数 
public static const MEDIA:String = "media"

言語バージョン: ActionScript 3.0
ランタイムバージョン: AIR 3.0

メディアのサウンドを再生するためのモード。モバイルでこのモードを使用すると、サウンドはスピーカーフォン経由で出力されます。ヘッドホンが接続されている場合は、スピーカーフォンが無効になります。オーディオの動作と優先度を決定する場合、オペレーティングシステムではメディアオーディオのデフォルト値を使用します。

この設定がデスクトップまたはテレビに特別な影響を与えることはありません。

これはデフォルトの設定です。

    

VOICE

定数 
public static const VOICE:String = "voice"

言語バージョン: ActionScript 3.0
ランタイムバージョン: AIR 3.0

ボイスオーディオを再生するためのモード。このモードを使用すると、デフォルトでは、オーディオが通話用のイヤホン経由で出力されます。ヘッドホンが接続されている場合は、イヤホンが無効になります。SoundMixer.useSpeakerphoneForVoice プロパティが設定されている場合、サウンドはスピーカーフォンに送られ、イヤホンとヘッドホンは無効になります。オーディオの動作と優先度を決定する場合、オペレーティングシステムではボイスオーディオのデフォルト値を使用します。

AudioPlaybackMode.VOICE の使用は最小限に抑えるようにし、音声呼び出しが終了して他のアプリケーションがメディアモードで再生できるようになった後は、できるだけ早く AudioPlaybackMode.MEDIA モードに切り替えるようにしてください。

この設定がデスクトップまたはテレビに特別な影響を与えることはありません。
